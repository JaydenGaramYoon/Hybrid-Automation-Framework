# Documentation layout

Topic folders keep CI notes, triage references, and AI prompt assets separate. Add new markdown here instead of scattering files at the repo root.

| Path | Purpose |
| --- | --- |
| `docs/ci/` | Jenkins, pipeline, and CI troubleshooting notes |
| `docs/triage/` | Triage rules, classification notes, runbook-style references |
| `docs/ai-prompts/` | Prompt text, schema notes, or MCP-related documentation |
| `docs/test-cases/` | Per-`tc_id` scenario markdown; conventions in **root `README.md`** (*Manual test cases*), index in `docs/test-cases/README.md` |
| `docs/assets/` | Images and static assets (e.g. `ci-screenshots/` for README or portfolio captures) |
