## Objective

After placing an order with the identity, product, shipping country, and any optional payment fields defined for this case, the user sees an order-completion message that matches the expected text stored for this row.

## Preconditions

- The user can sign in with the credentials on the row.
- The configured product is available to add to the cart.
- The user has reached the checkout flow with that product (for example: logged in, product in cart, checkout screen open).

## Test data

Authoritative field values are in **`testData/CheckOut.xlsx`**, Sheet **Sheet1**, row **`tc_id`** = **TC012**. Do not duplicate cell values here; open the file when executing or reviewing the case.

## Steps

1. Sign in with **email** and **password** from the row.
2. Add **`product`** to the cart, open the cart, and navigate until the **checkout** screen is shown (path depends on the app: catalogue → cart → checkout).
3. On checkout, enter **first name** and **last name** from the row; enter **email** if the form asks for it separately.
4. Select **country** and complete shipping fields as required; enter **credit_card_number**, **cvv**, and **coupon** only when the row has values or the app requires them.
5. Click **Place order** / **Pay** / **Submit** (the control that finalizes the purchase).
6. On the order **confirmation** screen, read the main headline or body message and confirm it matches **`expected_message`** on the row.

## Expected result

- The confirmation message shown after a successful order matches **`expected_message`** from the sheet (subject to application wording).
