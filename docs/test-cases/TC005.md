# TC005 - As a user signing up, I want the form to reject an invalid email format so that I cannot submit unusable contact details.

| Field | Value |

| **Title** | As a user signing up, I want the form to reject an invalid email format so that I cannot submit unusable contact details. |

## Objective

When the email value is **not a valid email** (for example domain only, per **`invalid_email`** on the row), the email field shows validation text that matches **`expected_message`**.

## Preconditions

- The sign-up screen is available.

## Test data

Authoritative field values are in **`testData/SignUp.xlsx`**, Sheet **Sheet1**, row **`tc_id`** = **TC005**. Use **`invalid_email`**, **`focus`**, and **`expected_message`** as defined there.

## Steps

1. Open the **sign-up** page.
2. Enter **first name**, **last name**, **occupation**, and **gender** from the row.
3. In the **email** field, enter **`invalid_email`** exactly as stored on the row (invalid format for this test).
4. Enter **phone** and **password** with otherwise valid values so that validation focuses on email (per row / app rules).
5. Click **Register** / **Sign up**.
6. Confirm the **email** field shows an error or hint whose text matches **`expected_message`** on the row (registration must not succeed with this email).

## Expected result

- The email field error or hint matches **`expected_message`** from the sheet (subject to application wording).
