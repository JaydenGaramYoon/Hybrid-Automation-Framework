# TC006 - As a user signing up, I want the form to enforce password complexity rules so that my password meets security requirements.

## Objective

When the password does not meet policy (for example letters only, per **`invalid_password`** on the row), the password field shows validation text that matches **`expected_message`**.

## Preconditions

- The sign-up screen is available.

## Test data

Authoritative field values are in **`testData/SignUp.xlsx`**, Sheet **Sheet1**, row **`tc_id`** = **TC006**. Use **`invalid_password`**, **`focus`**, and **`expected_message`** as defined there.

## Steps

1. Open the **sign-up** page.
2. Enter **first name**, **last name**, **occupation**, **gender**, **email**, and **phone** from the row with valid values (or acceptable test data per Preconditions).
3. In the **password** field, enter **`invalid_password`** from the row (value that violates the app’s password rules).
4. Click **Register** / **Sign up**.
5. Confirm the **password** field shows validation text matching **`expected_message`** on the row and the account is not created with this password.

## Expected result

- The password field error or hint matches **`expected_message`** from the sheet (subject to application wording).
