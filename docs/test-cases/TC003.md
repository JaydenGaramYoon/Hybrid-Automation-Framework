## Objective

Signing in with a **valid email** but a **wrong password** (per the row) shows an error message that matches **`expected_message`**.

## Preconditions

- An account exists for the email on the row; the password used in the test is the incorrect one defined on the row (**`invalid_password`** / **`focus`** as applicable).

## Test data

Authoritative field values are in **`testData/LogIn.xlsx`**, Sheet **Sheet1**, row **`tc_id`** = **TC003**.

## Steps

1. Go to the **login** screen.
2. In **email**, enter the value from **`email`** on the row (valid account).
3. In **password**, enter the value from **`invalid_password`** on the row (wrong password; **`focus`** should indicate password is under test).
4. Click **Sign in** / **Login**.
5. Confirm sign-in fails and the shown message matches **`expected_message`** on the row.

## Expected result

- The message shown matches **`expected_message`** from the sheet (subject to application wording).
