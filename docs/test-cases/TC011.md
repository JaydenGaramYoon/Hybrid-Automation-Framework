## Objective

After signing in and adding the **catalogue product** from the row to the cart, checking for a **different** product label (**`invalid_product`**) confirms it does **not** appear as a cart line (for example it was never added or does not exist in the catalogue).

## Preconditions

- Same sign-in and add-to-cart setup as for TC010: add the row’s **`product`** to the cart, then evaluate presence of **`invalid_product`**.

## Test data

Authoritative field values are in **`testData/Cart.xlsx`**, Sheet **Sheet1**, row **`tc_id`** = **TC011**. Use **`product`**, **`invalid_product`**, **`invalid_reason`**, and **`expected`** as defined there.

## Steps

1. Sign in with **email** and **password** from the row.
2. Add **`product`** from the row to the cart (the real catalogue SKU).
3. Open the **Cart** page.
4. Look for a line item whose name matches **`invalid_product`** on the row (a label that was **not** added in step 2).
5. Confirm no line item uses **`invalid_product`** as the product name. The cart should still show **`product`** from step 2 if the app lists added items.

## Expected result

- **`invalid_product`** from the sheet is **not** present in the cart (subject to how the application matches product names).
