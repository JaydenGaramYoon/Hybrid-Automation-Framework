## Objective

After signing in and adding the configured **product** to the cart, the cart view lists that same product (name or line item as defined by the application).

## Preconditions

- The user can sign in with the credentials on the row.
- The product on the row can be found and added from the catalogue.

## Test data

Authoritative field values are in **`testData/Cart.xlsx`**, Sheet **Sheet1**, row **`tc_id`** = **TC010**. Do not duplicate cell values here; open the file when executing or reviewing the case.

## Steps

1. Sign in with **email** and **password** from the row.
2. Go to the product catalogue (or shop) and find the item named **`product`** on the row.
3. Add that product to the cart (default quantity unless the sheet specifies otherwise).
4. Open the **Cart** page (or cart panel).
5. Verify a line item for **`product`** appears (same product name or label as when it was added).
6. If the row defines **`expected`** for reporting, note it; the pass/fail rule is that the product is visibly present in the cart.

## Expected result

- The cart contains the **product** defined on the sheet (subject to how the application displays line items).
