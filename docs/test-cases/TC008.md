## Objective

When the phone value contains a **non-numeric** character at the end (per **`invalid_phone_number`** on the row), the phone field shows validation text that matches **`expected_message`**.

## Preconditions

- The sign-up screen is available.

## Test data

Authoritative field values are in **`testData/SignUp.xlsx`**, Sheet **Sheet1**, row **`tc_id`** = **TC008**. Use **`invalid_phone_number`**, **`focus`**, and **`expected_message`** as defined there.

## Steps

1. Open the **sign-up** page.
2. Enter **first name**, **last name**, **occupation**, **gender**, **email**, and **password** from the row with valid-format values.
3. In the **phone** field, enter **`invalid_phone_number`** from the row (includes a disallowed character for this scenario, e.g. trailing symbol).
4. Click **Register** / **Sign up**.
5. Confirm the **phone** field shows validation text matching **`expected_message`** on the row.

## Expected result

- The phone field error or hint matches **`expected_message`** from the sheet (subject to application wording).
